docker网络模型初探

发表于 2018-04-09 | 分类于 golang ， docker |

前言

网络模型是操作系统的一大板块，docker 也针对各种应用场景推出了对应的网络模型，已经能够满足我们大多数需求

桥接网络

Host 网络

Overlay 网络

Calico 网络

自定义网络模型

子网介绍

bridge网络只能指定一个子网，而overlay网络支持多个子网

容器新增端口映射

方法1

使用 iptables 来添加端口映射

方法2

使用 docker commit containerid 保存容器为镜像，然后重新启动镜像，带上端口映射

补充

1.在 docker for mac 中，因为 docker 的真实宿主机是 hyperkit，因为当你使用 host 网络模型的时候，他共享的是宿主机的网络模型，因此你在容器中暴露出的服务，在 host 主机中是没办法访问到的

2.在桥接网络模型中的 -p 选项，映射出来的端口在宿主机中是可以访问的，因为宿主机中会把网络端口给代理出来
