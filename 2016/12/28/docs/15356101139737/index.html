Git 使用攻略

发表于 2016-12-28 | 分类于 git |

前言

我们经常提到git，github，说到git协议，到底都是一些什么呢？其中git是一种用作版本控制的工具，通俗地说就是用来管理代码的工具，github是一个网站，用来存放代码的，而git协议则是git工具私有的协议，用作传输代码。这里来撸一撸git相关的原理和命令。

git支持的协议汇总和比较

我们目前使用的git支持的主要协议有SSH、GIT、HTTP、HTTPS、FTP、FTPS、RSYNC及本地协议，其中本文注意围绕ssh协议进行总结，因为ssh协议是目前应用起来最简单，也最广泛的协议。 协议规范如下:

协议名称 语法格式 说明
SSH协议（1） ssh://[user@]example.com[:port]/path/to/repo.git/ 可在URL中设置用户名和端口。 默认端口22。
SSH协议（2） [user@]example.com:path/to/repo.git/ 更为精简的SCP格式表示法，更简洁。但是非默认端口需要通过其他方式（如地址别名方式）设定。
GIT协议 git://example.com[:port]/path/to/repo.git/ 最常用的只读协议。
HTTP[S]协议 http[s]://example.com[:port]/path/to/repo.git/ 兼有智能协议和哑协议。
FTP[S]协议 ftp[s]://example.com[:port]/path/to/repo.git/ 哑协议。
RSYNC协议 rsync://example.com/path/to/repo.git/ 哑协议。
本地协议（1） file:///path/to/
本地协议（2） /path/to/repo 和file://格式的本地协议类似。但有细微差别。 例如克隆时不支持浅克隆，且采用直接的硬连接实现克隆。

具体各种协议的优势和差别可以参考链接: https://git-scm.com/book/zh/v2/%E6%9C%8D%E5%8A%A1%E5%99%A8%E4%B8%8A%E7%9A%84-Git-%E5%8D%8F%E8%AE%AE

git的ssh登录相关

git查看本地key情况

cd ~/.ssh 然后 ls 可以查看是否有密钥文件

如果你的非对称加密算法用的是rsa，那么你的git默认目录下会有两个文件

id_rsa 和 id_rsa.pub

其中id_rsa是你的密钥,id_rsa.pub是你的公钥，你需要把公钥传到github上，然后自己保留密钥，和

github进行通信的时候获取的数据会在github用公钥来进行加密，然后在本地用密钥来进行解密。

如果以上文件不存在，那就需要自己先生成公钥和密钥，做好相应配置

git修改本地key密码

ssh-keygen -p

git生成公钥，密钥

生成公钥 并保存在~/.ssh／id_rsa.pub

ssh-keygen -t rsa -b 4096 -C “hbw123@qq.com“

就会生成id_rsa和id_rsa.pub两个密钥文件

复制.pub文件中的公钥字符串并保存在服务端

查看信息

显示有变更的文件

$ git status

显示当前分支的版本历史

$ git log

显示commit历史，以及每次commit发生变更的文件

$ git log –stat

搜索提交历史，根据关键词

$ git log -S [keyword]

显示某个commit之后的所有变动，每个commit占据一行

$ git log [tag] HEAD –pretty=format:%s

显示某个commit之后的所有变动，其”提交说明”必须符合搜索条件

$ git log [tag] HEAD –grep feature

显示某个文件的版本历史，包括文件改名

$ git log –follow [file] $ git whatchanged [file]

显示指定文件相关的每一次diff

$ git log -p [file]

显示过去5次提交

$ git log -5 –pretty –oneline

显示所有提交过的用户，按提交次数排序

$ git shortlog -sn

显示指定文件是什么人在什么时间修改过

$ git blame [file]

显示暂存区和工作区的差异

$ git diff

显示暂存区和上一个commit的差异

$ git diff –cached [file]

显示工作区与当前分支最新commit之间的差异

$ git diff HEAD

显示两次提交之间的差异

$ git diff [first-branch]…[second-branch]

显示今天你写了多少行代码

$ git diff –shortstat “@{0 day ago}”

显示某次提交的元数据和内容变化

$ git show [commit]

显示某次提交发生变化的文件

$ git show –name-only [commit]

显示某次提交时，某个文件的内容

$ git show [commit]:[filename]

显示当前分支的最近几次提交

$ git reflog

常见问题

1.git ssh克隆项目为什么经常会碰到需要输出用户名密码登录?

因为本地的密钥外加了一个密码，我的理解是二重加密，如果需要更改这个密码请查看 git修改本地key密码

2.git公钥密钥加密的应用场景是如何实现的,什么时候加密，什么时候解密？客户端update需要加密吗？

git的加密算法，主要是用来做ssh登陆验证的，和传输代码加密没有啥关系，只是为了你push代码前验证下你的身份

服务器先获取一组随机数A，把用公钥进行非对称算法加密得到B，然后通过网络将B传输给用户，用户获取到B之后，会用相对于的密钥进行解密，这样就得到了C,然后把C传给git服务器，git服务器比较这两者如果是一样，就不需要用户做身份验证了
