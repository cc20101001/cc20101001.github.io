python库分析之socket编程和系统调用

发表于 2016-12-28 | 分类于 python ， python库分析 |

前言

socket 并不是一种协议而是进程间通信的一种常用方法，属于ISO模型中传输层的编程接口，是web服务器的基石，套接字连接需要五个基本元素：

1.连接使用的协议

2.本地主机的IP地址

3.本地进程的协议端口

4.远程主机的 IP地址

5.远程主机的协议端口

这篇文章主要通过介绍 python 中 socket 库的用法来了解socket

套接字说明

套接字可以简单的理解为传输层的编程接口，提供给应用层来直接调用网络层的TCP和UDP协议等服务，甚至自定义网络层协议。通过socket可以区分不同的应用程序和传输层的对应关系，实现网络通信的数据传输和并发服务

套接字格式

socket(family, type[,protocal]) 使用给定的套接族，套接字类型，协议编号（默认为0）来创建套接字

套接族类型

套接族类型 描述 补充说明
socket.AF_UNIX(1) 同一台机器上进程间内部通信的时候经常使用 通常会有一个本地socket文件，进程间通过他来通信
socket.AF_INET(2) 使用范围更广，服务器与服务器之间的ipv4网络通信 也可以同一台机器的进程间通信
socket.AF_INET6(30) 服务器与服务器之间的ipv6网络通信 ipv6通信

套接字类型

套接字类型 描述 补充说明
socket.SOCK_STREAM 基于TCP的流式socket通信 流式传输
socket.SOCK_DGRAM 基于UDP的数据报式socket通信 数据报文传输
socket.SOCK_RAW 原始套接字，普通的套接字无法处理ICMP、IGMP等网络报文，而SOCK_RAW可以；其次SOCK_RAW也可以处理特殊的IPV4报文；此外，利用原始套接字，可以通过IP_HDRINCL套接字选项由用户构造IP头 原始套接字传输，需要有超级管理员权限
socket.SOCK_SEQPACKET 可靠的连续数据包服务 连续服务包传输

总结

socket非常重要，是传输层的编程接口，是现在许多web系统的基石

tcp 发起端涉及到的四个系统调用

1.拿到句柄 2.建立连接 3.发送通知 4.拿到会话句柄 5.收到消息 6.发送消息 7.关闭句柄

tcp 接收端涉及到的系统调用

1.拿到句柄 2.绑定端口 3.监听队列 4.接收消息 5.关闭句柄

udp 发起端涉及到的系统调用

1.获取句柄 2.绑定端口 3.端口监听 4.获取数据 5.关闭句柄

udp 接收端涉及到的系统调用

1.获取客户端句柄 2.发送数据 3.关闭句柄

socket 断开连接场景

通常我们在服务端用 recv 数据的长度来判定客户端数据是否发送完毕，如果发送完了就 close 掉服务端的socket，但如果是客户端先 close，服务端就会出现 CLOSE_WAIT 状态，没法 close

tcp服务端需要获取两次句柄，但是udp只需要获取一次句柄，因为tcp服务端还需要支持给客户端发送消息，所以需要单独获取句柄
