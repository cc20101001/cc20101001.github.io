前言

HTTP2 以及它的特性和优势

HTTP/2 试图从以下几个方面，解决 HTTP/1.1 中的几个历史遗留问题：

HTTPS 加密认证原理

https 能够保证安全因为他有两个核心，一个是传输数据加密，另一个是证书，数字证书存在的意义是为了解决公钥颁发过程中的安全性问题（实际上是把锅丢给了 CA，让 CA 来最大限度负责认证过程中的安全性，如果CA也没法保障，那么TLS就玩不下去了，事实是还是有漏洞啊）

SSL证书相关

SSL 证书申请根据类型不一样需要的材料不一样。SSL 证书根据身份验证方式分类有四种，根据等级分类有三种，具体介绍如下

SSL证书身份验证

SSL证书身份证验证有四类

Class 1：一级，只验证Email

Class 2：二级，验证Email和验证个人身份证明文件

Class 3：三级，验证Email、验证单位身份验证文件、第三方数据库核实

Class 4：四级，全球统一标准的扩展验证标准，验证Email、验证单位身份证明文件、验证申请人身份证明文件、第三方数据库核实

SSL证书等级

SSL证书没有所谓的”品质”和”等级”之分，只有三种不同的等级

DV是 Domain Validation 的缩写，意思就是对网站域名所有权进行验证（域名认证）。CA 认证机构会向域名持有者的邮箱发送相应的邮件，以确认证书和域名的所有权关系。其特点是简单快捷，价格便宜，缺点是无法保证网站经营者的身份，因此一般仅用来提供数据加密的功能，属于 Class 1 验证的证书。

OV是 Organization Validation 的缩写，这种证书在颁发的时候会对网站所有单位的身份和域名的所有权进行证实行验证（域名认证+单位认证），所以一般电子商务类的网站往往会做 OV 的认证。价格当然也会比较昂贵，证书颁发周期也会比较久，属于 Class 2 或者 Class 3 验证的证书。

EV是 Extended Validation 的缩写，也是最严格的身份验证，此证书审核证书申请人对域名的所有权，以及详细的企业/组织相关信息审核（域名认证+更详细的单位认证），当用户在访问通过 EV 认证的网站时候，浏览器的显示为绿色，当然价格也是相当的昂贵，属于 Class 4 验证的证书。

SSL/TLS

SSL 是洋文 “Secure Sockets Layer” 的缩写，中文叫做 “安全套接层”，到了1999年，SSL 因为应用广泛，已经成为互联网上的事实标准。IETF 就在那年把 SSL 标准化。标准化之后的名称改为TLS（是“Transport Layer Security”的缩写），中文叫做 “传输层安全协议”，后来我们并称为 SSL/TLS 协议

理论和效果简单做了介绍，现在开始试用了，我们要想能使用HTTP2，而在nginx里面HTTP2是基于HTTPS的（还有其他方法只用HTTP2），所以先需要搞定HTTPS，HTTPS 证书生成有两步

如果只为了自己测试，只需要自己生成证书请求文件和密钥就可以了，然后去自己的CA认证获取证书。如果还需要在互联网上让其他人访问，建议去第三方 CA 购买数字证书，提高网站证书的可信度

这里说下 Linux 系统怎么通过 openssl 命令生成证书

1.生成一个 des 标准密钥（这里才用des3对称加密算法）

他会要求输入密钥密码。不推荐输入。如果以后要给nginx使用。每次 reload nginx 配置时候都要你验证私钥密码

2.根据这个密钥文件生成证书签名请求文件 Certificate Signing Request（CSR），CSR 是为了去 CA 做认证的，不用带上密钥，更安全

1和2可以结合起来为一步，生成 pkcs8 格式的密钥和 CSR 文件

3.提交 CSR 给 CA 获取数字证书认证，颁发证书

CSR 文件必须有 CA 的签名才可形成数字证书。可将此文件发送到 verisign 等 CA 验证，但要交一大笔钱。如果只是为了内部测试，何不自己做 CA 呢，openssl 也是支持的，只需要按照屏幕提示，一步步输入要求的个人信息即可

nginx安装

如果在 nginx 里面做 http2 部署，需要我们的nginx服务器支持 ssl，prec，http2 模块，因为是mac环境，我采用brew安装，命令如下，仅供参考:

brew install nginx-full --with-lua-module --with-http_ssl_module --with-pcre --with-ipv6 --with-mail --with-http2 --conf-path=/usr/local/etc/nginx/nginx.conf

如果你用 nginx 自己编译，那么在 ./configure 中加入：–with-http_v2_module ，如果没有 SSL 支持，还需要加入 –with-http_ssl_module（因为http2依赖于https），–with-http_stub_status_module
