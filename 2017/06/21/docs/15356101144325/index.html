变量的生命周期

Nginx 变量的创建和赋值操作发生在全然不同的时间阶段

1.Nginx 变量的创建只能发生在 Nginx 配置加载的时候，或者说 Nginx 启动的时候

2.而赋值操作则只会发生在请求实际处理的时候

3.Nginx 变量的生命期是不可能跨越请求边界的，不同请求间的变量不能相互修改

这意味着不创建而直接使用变量会导致启动失败，同时也意味着我们无法在请求处理时动态地创建新的 Nginx 变量

变量的作用域
