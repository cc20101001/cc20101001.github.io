python模块分析之concurrent.futures-线程池，进程池

发表于 2017-06-11 | 分类于 python ， python库分析 |

前言

在 python 里面前人封装了许多不错的轮子，非常方便我们调用，这里介绍一个 python 标准库 concurrent.futures，里面封装了一些高性能的 API 来执行异步任务，我们这里主要介绍他的 ThreadPoolExecutor 线程池和 ProcessPoolExecutor 进程池，我们能够将任务塞到 pool 里面，pool会自动完成调度，分配任务给相应的 threads（process）并且执行，非常方便

进城池（ProcessPoolExecutor）

进程池和线程池比较类似，提供一样的接口，比较而言，进程池更适合 CPU 密集型任务，而线程池更适合 IO 密集型任务，ProcessPoolExecutor 底层调用了 multiprocessing 模块，并且不受 GIL 锁的限制（因为多进程，大家的解释器又不一样）

concurrent.futures.Future

该对象可以对注册回调函数的 Future 实例进行操作以及状态进行检测，有如下方法可以进行调用

cancel()

Attempt to cancel the call. If the call is currently being executed and cannot be cancelled then the method will return False, otherwise the call will be cancelled and the method will return True.

取消Future实例的执行

cancelled()

Return True if the call was successfully cancelled.

查看Future实例状态

running()

Return True if the call is currently being executed and cannot be cancelled.

done()

Return True if the call was successfully cancelled or finished running.

查看Future实例是否执行完

result(timeout=None)

Return the value returned by the call. If the call hasn’t yet completed then this method will wait up to timeout seconds. If the call hasn’t completed in timeout seconds, then a concurrent.futures.TimeoutError will be raised. timeout can be an int or float. If timeout is not specified or None, there is no limit to the wait time.

If the future is cancelled before completing then CancelledError will be raised.

If the call raised, this method will raise the same exception.

这个函数比较关键，返回 Future 回调函数执行的返回值

总结

1.池都是 concurrent.futures.Executor 的子类

2.定义池的时候返回的是一个 concurrent.futures.Executor 对象，可以通过该对象往池里面注册回调函数，得到Future类或者他的迭代器

3.注册回调函数之后返回的是 concurrent.futures.Future 对象，该对象可以对注册回调函数的 Future 实例进行操作以及状态进行检测

4.wait() 和 as_complate() 比较类似，但是更灵活
