npm安装和使用

发表于 2017-03-23 | 分类于 js ， 包管理器 |

前言

npm本来是node.js的包管理工具，但随着js这几年的蓬勃发展，现在的npm已经成了几乎所有跟JS相关的工具和软件包的管理工具了，并且还在不断发展完善中，这里撸一撸npm的安装和使用

npm的安装

最简单安装 npm 的方式就是去安装一个 nodejs，下载地址: https://nodejs.org，当然你也可以使用源码包自己编译的方式去安装

npm 安装其他包

npm安装其他包有两种形式：

1.在当前目录（工程）安装，一般真对某些特定工程，做特定安装，安装的包会放在当前工程的node_modules目录

2.在全局安装（一般都是一些命令行工具），需要在全局使用,安装的包会放在root的lib目录，比如我的环境是mac，路径就是 /usr/local/lib/node_modules 也可能是 /usr/lib／node_modules 目录 ，当然你也可以用命令来查看。

npm包的缓存管理

npm安装包的时候，会先去缓存查看这个包是否有过下载，如果已经下载过就直接拿来安装，但有时候这些包很老了，就需要我们去清理npm的缓存安装包

npm工程的初始化

在当前目录下初始化出一个npm工程，他就有了 node_modules 目录和 package.json 文件,我们就可以在当前目录安装其他包了，但是有许多参数需要填写，当然也提供给了我们快速初始化命令 npm init --yes

npm package.json文件说明

package.json简介

package.json文件注意来存放npm的一些配置信息，有了 package.json 以后，在为项目安装包的时候，可以把安装的包保存到 package.json 文件里的 dependencies(–save) 或者 devDependencies(–save-dev) 里面，这样做可以更容易跟别人分享我们做的项目。

继续 cat package.json 会发现依赖已经添加进来了（因为用的是–save所以添加到了dependencies中）

–save-dev和–save的区别

–save-dev 是你开发时候依赖的东西，–save 是你发布之后还依赖的东西。

比如，你写 ES6 代码，如果你想编译成 ES5 发布，那么 babel 就是devDependencies。 如果你用了 jQuery，由于发布之后还是依赖jQuery，所以是dependencies。

但是在 npm 里面除了二进制的依赖，似乎也不用区分是不是dev。 因为使用npm就是自己编译的意思，而不使用npm直接拿编译后的版本的，这些依赖项也看不到。
