博客写作一周年有感

发表于 2017-08-10 |

笔尖的思考

我的第一篇博文是16年8月19号写的，离今天也快接近一年了，在这一年里通过写博客，反思一些技术细节和学习方法，收益匪浅

认识自己

我自己算是一个比较喜欢折腾的家伙，五花八门的东西都折腾过，从AI模式识别，到云计算，虚拟化，等等，都有过一些了解，慢慢的我发现几个问题需要我去解决以后才能提高自己

克服忘记

回顾以前学习的一些知识点，如果长时间不用，会以非常快的速度忘记掉，从这个角度来说，写博客，对自己学习思考的轨迹做一些记录，方便以后回顾，反思，快速拾起记忆的碎片，还是非常有帮助的

技术深度

做技术很忌讳了解不深入，大学时期和工作之初，我也算是一个猴子掰玉米的典型案例，了解的东西太多了，但是总感觉深入度不够，没能达到我的预期，写博客的时候一方面帮我整理思路，如何才能将某个知识点最直观的表达出来，以及该知识点和哪些知识点是有关联的。让自己写的东西形成一个体系，就像塞积木一样，把每一块知识点塞到技术体系上合适的位置

构建自己的技术体系

技术的世界千奇百怪，每一个细分的领域都有自己的专家，而我在这个世界该如何立足？

构建自己的技术体系尤为重要，我也尝试过去了解现在的前端技术，成为一个传说中的全栈工程师，但是发现我现在根本没那么多精力去学精通，因为前端老的技术非常多，新的技术，又层出不穷，研究来研究去，研究了一个框架，还没吃透，没过两年这个框架又过时了，如果长期做前端应该可以适应这种节奏，但是做后端也想完全精通前端，是非常难的，起码对我而言，我目前是做不到的，我只能够了解了解，然后学好一些前端基础知识，比如 dom，http协议，css，html，js。vue，angularjs 啥的了解一下大致用法和原理就行了，说不准哪天需要写个简单的应用，这是我对我自己的要求

博客主题选择

博客主题选择适合我选择了 hexo 的 next，原因挺简单：

1.wordpress 为首的传统博客系统我不想采用（系统过于臃肿，我需要的是一个非常简洁的系统），hexo作为新型系统，设计思想简洁，可扩展性高，是我考虑的首选

2.必须对markdown支持友好（完美）

3.能够用 MWEB 写完后直接推送到公网服务器，操作需要非常方便，不需要登录公网服务器，不需要排版，不需要点击发布按钮啥的

4.主题支持自定义，最好在我的能力范围之类，所以尽量用js，css或者scss，最好不需要我去了解系统特定的模板语言

5.博客换主题需要非常简单，并且有丰富的主题可以选择

综合以上诸多考虑，hexo 是最完美的解决方案 https://hexo.io/

博客部署方式

我现在博客部署在 od 洛杉矶机房上，速度还行吧（服务器开启了 bbr 加速和 ssr 的 udp 加速，科学上网杠杠的）

1.在 mac 本地用 markdown 编辑器（mweb）编写文章

2.用 hexo 生成静态博客

3.hexo 集成了 github 直接把生成的静态博客推送到 github

4.openresty 的 html 目录下 clone 了 github 上静态博客代码，直接pull到本地（也可以用定时任务自动拉取）

5.在vps上部署了 openresty 环境并且配置了http2环境

6.博客https证书用的 Let’s Encrypt certificates 的 DV 证书

7.集成 very nginx
