golang学习笔记（1）

发表于 2017-08-02 | 分类于 golang ， golang学习笔记 |

前言

这个系列主要系统介绍 golang 基础知识学习，学习过程中某些地方也会和 python 进行对比，加深理解，巩固记忆，学 golang 最好有一点 c 的底子，因为他和 c 还挺像。第一节里主要包含了 golang 的一些基本知识

1.基本开发环境

2.标识符

3.基本数据类型，变量，常量，常量组，枚举，数据类型转换

4.运算符

5.条件控制语句，if else，for，goto，switch，select

6.函数，普通函数，闭包，延时函数，延时执行函数，函数的调用，值传递，引用传递，方法

7.注释，格式化输出

基本开发环境

这次学习我用的是 jetbrains 的 Golang1.0 EAP 来 coding 一些代码，golang 安装包直接可以去官网下载，我的环境很久前就搭建了，搭建其实非常方便，我也就不详细记录环境搭建过程了，可以参考之前写过的文章：go语言开发环境搭建

标识符

在 golang 里面标识符就是那些给变量，常量，类型等命名的字符串，他的第一个字符不能为数字，标识符里面有一个非常特殊的符号 _，通常作为忽略占位符使用，意思就是系统获取了返回值但是不允许被使用，避免编译器因检查某些变量未被使用而报错

数据类型转换

在 go 里面的数据类型转换强制要求使用显示类型转换，格式：type_name(expression)，type_name 为类型，expression 为表达式（可以为变量）

在 go 里面比较奇葩的不能将非 bool 类型结果当作 true/false 使用，即 if value 中的 value 一定是一个 bool 值

运算符

golang 运算符有许多种类，如算术运算符，关系运算符，逻辑运算符，位运算符，赋值运算符，其他运算符，里面和 python 一样的我就不多说了，重点介绍一下他的自增自减运算符，如 ++ --，他还真只是自增自减，但是返回的结果不能赋值（没有 k = t++）

条件控制语句

go 的条件控制语句和其他语言比较类似，有一样的 if，else，for，以及嵌套 switch，select，多了 goto 语法，可以改变函数执行流程，类似 python 中的 yield

函数的值传递与引用传递

函数参数分值传递和引用传递，接收参数为指针类型的是引用传递，对参数的修改可以改变原来传入参数的值，接收参数为普通数据类型的为值传递，是新分配的内存空间，对值的修改不会修改原有传入参数的值

方法

golang 的方法非常特别，在接触 golang 前，我以为方法和函数是同一东西的两种叫法，实际上在 golang 中是两个完全不一样的概念（回头看了下 python，也有方法和函数的区分）

我们定义了一个变量 myint 类型为 int，然后定义了一个方法，方法和函数的区别在于 func 之后跟着的是函数名，还是一个对象，意思是这个函数属于哪个对象（并且返回值也是一个对象，有点绕，有点像给实例化后的对象赋值）

golang 中的方法我们通常用在结构体和自定义数据类型的场景中

这里有一个小区别，对于 my_func_1 和 my_func_2，用指针传递的是引用传递，另一个是值传递，引用传递会改变原来传入参数的值，而值传递不会（和 c 语言类似）

方法的重载

在 golang 中方法中只要绑定的是不同的对象，他是没有方法重载这个概念

Print: 普通输出

Println: 输出换行，每次输出完后默认加一个换行符 \n

Printf: 类似 c 语言的自定义格式化输出，最方便，我最喜欢

总结

golang 语法比较独特，也很怪异，既有静态语言 C 的特点，也有动态语言如 Python 的特点，虽然当时推出 golang 的目标是为了吸引 c++ 开发者，实际上倒吸引了不少 Python 开发者，如果非要找一个最像的语言，应该是 C 吧，把 golang 的语法当作 C 和 Python 的结合体来看，很容易懂
