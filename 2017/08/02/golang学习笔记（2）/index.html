数组

golang 因为有数据类型需要声明，所以数组相比 python 用起来也就复杂了一些，数组定义格式: var arr [n]type，用法和其他语言有点类似，有标识符，长度，数组对象类型，下标访问等特性，下面是数组的多种定义方式

如果定义的数组类型是 byte，那应该用 ‘’ 单引号来表示，如

my_byte := [2]byte{'a', 'b'}

引用类型

golang 中的引用类型，特指切片（slice），字典（map），通道（channel）这三种预定义类型，相比其他数据类型，引用类型拥有更复杂的存储结构，除了内存分配外，她们还需初始化一系列属性，诸如指针，长度，甚至包括哈希分布、队列等，毕竟接口更多，功能更强大

切片 slice

golang 的切片和 python 的有点像，不同点是 golang 是对数组中对象的引用，对切片中元素的修改会反过来修改原数组中元素，而 python 是重新生成了一个列表对象

我们用 slice_1 生成了一个切片对象 t，然后修改了切片对象 t 中的第一个元素，发现原 slice_1 也被修改了

字典（map）

go 中也提供了 python 一样的字典类型 map，不一样的是他使用起来多一步初始化的操作，需要先定义一个 map 对象，然后再初始化 map 对象，最后才是使用 map 对象，给它赋值。map 对象的访问和 python 一样，并且支持遍历访问

range 语法

range 用于在 for 循环中迭代数组，切片，字典，字符串等元素，数组和切片中返回索引和值，字典中返回 key 值，这段代码对数组进行了迭代，并对数组 nums 进行求和

在 golang 中，我们也可以用 range 语法对 map 对象进行遍历。其中的 i 为 map 对象的 key，里面的 value 可以用 data[i] 数组加下标的方式获取

指针

在 golang 中有指针的概念，指针和内存地址是不一样的，内存地址是内存中每个字节单元的唯一编号，而指针则是一个实例，指针会分配内存空间，相当于一个专门用来保存内存地址的整形变量，里面存储了某个对象内存地址

1.取址运算符 & 用于获取对象地址

2.指针运算符 * 用于间接引用目标对象

3.二级指针 **T 如包含包名则写成 *package.T

指针定义格式：var point_name *type，point_name 为 指针变量名称，type 为指针的类型，不同类型的指针指向不同类型的变量

接口类型

golang 不是传统意义上的面向对象编程语言，他没有类的概念，也没有继承的概念，接口类型（当然还有结构体类型）为他弥补了这一点的不足，通过接口类型他规定了类型的行为，并且通过接口可以实现函数的多态

type Namer interface { Method1(param_list) return_type Method2(param_list) return_type … }
