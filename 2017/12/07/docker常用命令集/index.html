docker常用命令集

发表于 2017-12-07 | 分类于 golang ， docker |

前言

最近公司的系统上了 docker，以后怕是各种应用都会走容器化，这里收集整理了一些 docker 常用命令集，方便日后回顾查看

查看 docker 容器，版本号信息

docker info

docker version

docker 启动容器命令之-run

启动命令 options 详情如下

容器启动模式-后台运行模式

这种模式使用的最多，让容器在后台运行，我这里另外加了两个参数，指定了容器名称并且做了一个端口映射，把容器内部的 80 端口映射到了外部环境 8000，当我们访问 8000 端口的时候请求会被代理到容器内部 nginx 服务的 80 端口

容器启动模式-普通模式

这种模式使用的也较多，容器log日志会直接在前台进行输出，方便查看定位问题

容器启动模式-调试模式

调试模式使用完后一般需要容器自动删除，这种对于容器构建调试，太方便了

容器启动模式-交互模式

交互模式允许在容器内部执行命令，实际开发过程中用的并不多，如果有需要和容器进行交互都是等容器运行起来后用 exec 命令来交互

常用简单命令

  • docker inspect: 查看镜像或者容器细节信息

  • docker commit CONTAINER_ID: 从容器创建镜像

  • docker save/load: 保存, 恢复镜像信息，从镜像导出为镜像

  • docker export/import: 保存，恢复容器，容器导出为容器

  • docker top CONTAINER_ID: 查看特定容器资源占用信息，mac 环境下还有一款 ctop，用起来很方便

  • docker cp CONTAINER_ID: PATH HOSTPATH: 从镜像中复制文件到宿主机

  • docker diff CONTAINER_ID: 查看容器相对镜像的文件变化

  • docker history IMAGE_ID: 查看镜像构建历史

  • docker tag IMAGE_ID [REGISTRYHOST/][USERNAME/]NAME[:TAG]: 给镜像打标签

  • docker exec 提供了与容器进行交互的方法，包括 ssh连接，程序运行等

  • docker attach 类似 exec 命令，不同的是中途退出会导致容器停止

详细请看 Dockerfile 镜像构建

docker 登陆镜像库

docker login -u test -p test xxx.dockhub.com

docker for mac 的一些吐槽

docker for mac 虽然方便了 macos 用户使用 docker，但还是不够完美

1.docker 底层的一些东西，如镜像，容器，分层等信息，都放到了虚拟机中，查看起来不方便，还需要连接到 HyperKit screen ~/Library/Containers/com.docker.docker/Data/com.docker.driver.amd64-linux/tty

ubuntu 容器数据放在了 /var/lib/docker 目录

2.你自己制作了一个 docker 容器，里面有 print 输出函数，在 macos 环境里面的显示有点问题（连接到 HyperKit 中也没法显示），也可能是我环境问题

docker ps

docker ps 列出了所有运行态容器，隐藏了非运行态容器，如果想查看所有容器，需要用 docker ps -a 指令

docker images

docker images 会列出所有的镜像，这里的镜像和我们通常所说的镜像有区别，每 pull 一次，称作为一个镜像，每一层镜像下面可能隐藏了多层镜像

docker rm

docker rm 命令会移除 docker 的可读写层，也就是移除容器

docker rmi

docker rmi 命令会移除 docker 的只读层，也就是移除镜像
