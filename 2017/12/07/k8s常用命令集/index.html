k8s常用命令集

发表于 2017-12-07 | 分类于 golang ， k8s |

前言

系统架构最近变更比较大，从最初的应用程序裸体部署，到中间的 docker 容器部署，再到现在的 k8s 集群管理，也是一步一个台阶，这里主要记录了一些 k8s 命令集，方便日后回顾，查看

获取 deploy 信息

最新版的k8s 1.9.x deploy 的配置非常的简洁，deploy 替代的 rc 作为副本控制器，更加方便，获取所有的 deploy 信息

kubectl get deploy –all-namespaces

获取所有命名空间的的pod

如果机器环境很多，我们需要给每一套环境分配一个命名空间，默认 kube-system 是 k8s 系统默认命名空间，获取所有命名空间 pod 的命令如下

kubectl get pod –all-namespaces

获取 services 的对外统一接口

k8s 对容器化编排的核心是 pod，pod 如果想对外网提供服务，需要一层 service 的包装，好处是做了一层负载均衡，一个 service 后面可以挂多个 pod

kubectl get services

获取 pod 提供服务的web接口

endpoints 是 pod 提供服务的接口

kubectl get endpoints(ep)

设置 kubectl 运行环境 context

kubectl config use-context minikube

kubectl config use-context docker-for-desktop

pod删除

强制删除某些状态异常的pod（状态为Terminating），系统默认杀掉pod的延时执行时间是30s，这里设置为立刻执行

kubectl get pod | grep Terminating | awk ‘{print $1}’| xargs -n1 -I {} kubectl delete pod {} –grace-period=0 –force

configmap

这里放置了 k8s 的一些配置信息，可以把他当作 k8s 的环境变量设置地方，一处配置，多处调用

ingress，ingress-controller，nginx

k8s 如果想对外提供服务，是以 ingress 的形式提供的

查看日志

根据标签筛选pod，查看所有信息

kubectl logs -lqcloud-app=hcm-celery-other -n release

使用 journalctl 查看服务日志

etcd，kube-apiserver，kube-scheduler，kube-controller-manager，kubelet 都有对应的日志

journalctl -u etcd -f

交互式命令

命令和 docker 类似

kubectl exec -it xxxx-pod -n relase bash
