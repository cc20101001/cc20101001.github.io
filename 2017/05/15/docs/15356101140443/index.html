基础算法编程系列之-时间复杂度和空间复杂度篇

发表于 2017-05-15 | 分类于 算法 ， 基础算法编程 |

前言

同一问题可用不同算法解决，而算法的质量优劣将影响程序执行效率。算法分析的目的在于比较，选择合适算法，提高程序运行效率。一个算法的好坏主要从时间复杂度和空间复杂度来考虑

时间频度

百度百科：一个算法执行所耗费的时间，从理论上是不能算出来的，必须上机运行测试才能知道。但我们不可能也没有必要对每个算法都上机测试，只需知道哪个算法花费的时间多，哪个算法花费的时间少就可以了。并且一个算法花费的时间与算法中语句的执行次数成正比例，哪个算法中语句执行次数多，它花费时间就多。一个算法中的语句执行次数称为语句频度或时间频度。记为T(n)。算法的时间复杂度是指执行算法所需要的计算工作量

我：时间频度就是统计算法中语句执行次数的

时间复杂度

百度百科：前面提到的时间频度T(n)中，n称为问题的规模，当n不断变化时，时间频度T(n)也会不断变化。但有时我们想知道它变化时呈现什么规律，为此我们引入时间复杂度的概念。一般情况下，算法中基本操作重复执行的次数是问题规模n的某个函数，用T(n)表示，若有某个辅助函数f(n)，使得当n趋近于无穷大时，T（n)/f(n)的极限值为不等于零的常数，则称f(n)是T(n)的同数量级函数，记作T(n)=O(f(n))，它称为算法的渐进时间复杂度，简称时间复杂度

我：时间复杂度就是研究时间频度规律的，当n趋近于无穷大的时候它们两者相等

大O阶推导原则

大O阶推导三板斧

1.用常数1取代运行时间中的所有加法常数

2.在修改后的运行次数函数中，只保留最高阶项

3.如果最高项存在且不是1，则去除与这个项相乘的常数

常见算法时间复杂度比较

一般在没有特殊说明的情况下，我们分析比较的都是算法的在最坏状况下的时间复杂度。这里整理了一份常见算法时间复杂度的归档列表

时间频度 时间复杂度 术语
12 O(1) 常数阶
5log2n+20 O(logn) 对数阶
2n+3 O(n) 线性阶
2n+3nlog2n+19 O(nlogn) 线性对数阶
3n²+2n+1 O(n²) 平方阶
6n³+2n²+3n+4 O(n³) 立方阶
2^n O(2^n ) 指数阶

以下是常见函数时间复杂度比较关系：

O(1)<O(logn)<O(n)<O(nlogn)<O(n²)<O(n³)<O(2^n)<O(n!)<O(n^n)

空间复杂度

评估一个算法在运行时对内存的占用程度。我们一般用S(n)=O(f(n))来表示。
