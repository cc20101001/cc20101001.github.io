python虚拟环境模块virtualenv

发表于 2017-05-03 | 分类于 python |

前言

python版本问题一直是一个头大的问题，比如我现在系统里面有一个django1.3的版本，但是另外一个工程里面需要用到django1.5的版本，这个时候virtualenv 就派上用场了（如果你想在一个系统里面用到不同的模块版本，他是不会冲突的哟）

安装virtualenv

pip install virtualenv

创建虚拟环境

virtualenv [环境名]

eg:

virtualenv -p python3 env1

此时他会在系统里面创建一个名称为 env1 的目录，此目录下克隆了一份纯净版的 python3 环境

里面会有bin，lib，include三个目录，存放了环境变量所需要的一些软链，模块，解释器

启用虚拟环境

source ./bin/activate

退出虚拟环境

deactivate

删除环境

直接删除某个环境的目录

查看该虚拟环境安装了哪些模块

查看帮助

virtualenv -h

相关资料

https://virtualenv.pypa.io/en/latest/
