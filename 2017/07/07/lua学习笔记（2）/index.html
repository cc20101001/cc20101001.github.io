加载机制

对于自定义的模块需要放在特定的地方，lua的加载机制是会从当前路径下和 package.path 路径（也称作环境变量）下获取模块信息，并加载，如果lua文件里面找不到就会去C程序库里面找

加载调用C库

loadlib函数挺有意思，加载成功后他并没有运行这个函数，而是返回了一个函数对象，我们需要调用 f()，这个时候才是进行初始化

为防止动态链接库加载失败，我们可以断言 assert(loadlib(**))，让动态链接库加载成功后才返回

元表（metatable）

元表（metatable）是有一些特定属性的表（如index,newindex），是lua中一个非常重要的概念，有点类似python里面的元类（metaclass），在Lua table中我们可以访问对应的key来得到value值，但是却无法对两个table进行操作，设置元表允许我们改变table的行为，做一些有魔法特性的事情

元表中有两个很重要的函数：

setmetatable(table, metatable)：对指定table设置元表（metatable），如果元表（metatable）中存在__metatable键值，setmetatable会失败，设置元表格式如下

setmetatable({foo=2}， { index = other }) ，其中index后面的一多半是一个函数或者一个table

getmetatable(table):返回对象的元表

总结：

lua在查询表结构元素的时候，其实会有三个步骤

1.在表中查找，如果找到，返回该元素，找不到则继续

2.判断该表是否有元表，如果没有元表则返回nil，如果有元表则继续

3.判断元表有没有 index方法，如果没有则返回nil，然后去元表里面查数据，如果里面的元表还有index方法，则重复1，2，3；（在元表中找的过程中如果__index是一个函数，则返回该函数的返回值）

补充：

我们在使用 setmetatable(table, metatable)的时候可以直接传变量，他支持引用传递，并且不需要获取setmetatable的返回值，看起来更干练

lua中的协程（coroutine）

lua协程和线程比较类似，它拥有独立的堆栈，独立的局部变量，独立的指令指针，同时又与其它协程共享全局变量和其它大部分东西

协程和线程的主要区别在于，一个具有多个线程的程序可以同时运行几个线程，而协程是一个程序里面可能有多个协程，他们都在一个线程里面，需要彼此协同工作，在指定的时刻只有一个协程在运行，并且这个协程只要在明确的被要求挂起的时候才会被挂起，让其它协程继续运行。也有点类似同步的多线程，等待同一个线程锁

lua协程的基本语法

方法 描述
coroutine.create(f) 创建协程，返回coroutine
coroutine.resume(co [, val1, ···]) 重启coroutine，和create配合使用
coroutine.yield(…) 挂起coroutine并返回yield里面的对象
coroutine.status(co) 查看coroutine的状态（dead，suspend，running）
coroutine.wrap(f) 和create功能重复
coroutine.running() 返回正在跑的coroutine，一个coroutine就是一个协程，当使用running的时候，就是返回一个coroutine的协程号
