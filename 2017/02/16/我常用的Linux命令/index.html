lsof命令

lsof(list open files)是一个列出当前系统打开文件的工具。在linux环境下，任何事物都以文件的形式存在，通过文件不仅仅可以访问常规数据，还可以访问网络连接和硬件。所以如传输控制协议 (TCP) 和用户数据报协议 (UDP) 套接字等，系统在后台都为该应用程序分配了一个文件描述符，无论这个文件的本质如何，该文件描述符为应用程序与基础操作系统之间的交互提供了通用接口。因为应用程序打开文件的描述符列表提供了大量关于这个应用程序本身的信息，因此通过lsof工具能够查看这个列表对系统监测以及排错将是很有帮助的。

详见 http://www.cnblogs.com/ggjucheng/archive/2012/01/08/2316599.html

linux磁盘读写速度测试

写速度测试:

time dd if=/dev/zero bs=1024 count=1000000 of=/tmp/my_test

读速度测试:

time dd if=/tmp/my_test of=/dev/null bs=8k

压缩和解压文件

压缩文件夹名为 dir_to_be_tar 的文件到 save.tar.gz

tar -cvf ./save.tar.gz dir_to_be_tar

高比例压缩

tar -czf ./save.tar.gz dir_to_be_tar

解压 save.tar.gz 文件到当前目录

tar xvf data.tar.gz ./

tar -zxvf ./save.tar.gz

CPU 相关命令

查看 CPU 型号

cat /proc/cpuinfo | grep name | cut -f2 -d: | uniq -c

查看物理 CPU 个数

cat /proc/cpuinfo | grep “physical id” | sort| uniq| wc -l

查看每个物理 CPU 中 core 的个数(即核数)

cat /proc/cpuinfo | grep “cpu cores” | uniq

查看逻辑 CPU 的个数

cat /proc/cpuinfo | grep “processor” | wc -l

Linux DNS 解析问题

通常我们修改 Linux 都会修改 /etc/resolv.conf 文件，这样不好，因为每次重启网络后这个文件会被还原，在 /etc/resolv.conf 文件中本来就有相关注释

解决办法就是 DNS 直接配置到网卡设置中

集群部署通常需要做时间同步，方便管理

sed 命令

sed 命令很强大，可以替换文本如： sed -i s'/123/456/g' file_name 该命令把文件 file_name 中的 123 字符串替换为 456

在指定行范围内做替换

sed -i ‘2,5s#123#456#’ aa.txt

对于 aa.txt 文件在 2-5 行中把字符串 123 替换为 456

grep

正则表达式

大于500的正则表达式

不包含

grep -v “关键词” 筛选出不包含关键词的行

关闭交换分区

关闭分区前可以先查看分区列表: sfdisk -l

去掉或注释掉: /etc/fstab 中对应的 swap 分区

使用 swapoff /dev/sda2，关闭 /dev/sda2 分区

当SWAP占用率高达30%，对系统性能可能会有一定影响，所以在适当情况下，我们可以执行上述的两个命令刷新一次 SWAP（将SWAP里的数据转储回内存，并清空SWAP里的数据）

swapoff -a && swapon -a

目录，文件夹大小统计

查看，统计根目录下文件夹或文件大小，统计深度为1

du -h –max-depth=1 /

重启机器，关机

shutdown -t 0 其中 -t 是立刻关机的意思

reboot 机器重新启动
